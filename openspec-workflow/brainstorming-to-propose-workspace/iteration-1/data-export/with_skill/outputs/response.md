# 测试用例 2：数据导出功能（有 skill）

## 用户输入

> 帮我加个数据导出功能，要支持导出 Excel 和 PDF。表格用的是 ep-comp 的分页列表组件，数据量大概几万条。后端已经有 /api/export 接口了，只需要前端对接。项目里已经有 openspec 了。

---

## 阶段 1：brainstorming 需求探索

### 澄清问题

开始前需要确认几个关键点：

**1. 导出数据范围**

- A. 导出当前筛选条件下的全部数据（后端全量导出）
- B. 导出当前页的数据
- C. 用户自选范围（勾选行 / 指定条数）

几万条数据量，推荐 **A（后端全量导出）**，前端只需要传筛选参数和导出格式。

**2. 导出触发方式**

- A. 列表顶部工具栏加"导出"按钮，点击弹出格式选择
- B. 列表顶部直接放"导出 Excel"和"导出 PDF"两个按钮
- C. 下拉菜单包含多种导出选项

**3. 大文件导出的体验处理**

几万条数据的导出可能需要等待：
- A. 简单 loading，等接口返回后自动下载
- B. 后端异步生成，前端轮询下载状态，完成后推送下载
- C. 后端异步 + 前端消息通知

**4. 导出按钮的放置位置**

- A. 每个 ep-comp 列表页面都需要支持导出
- B. 只有特定页面需要导出
- C. 做成通用组件，按需引入

**5. 导出文件名规则**

- A. 固定文件名（如"数据导出.xlsx"）
- B. 带时间戳（如"用户列表-2026-04-16.xlsx"）
- C. 用户可自定义

---

### 假设用户回复：A、B、A、C、B

### 方案收敛

基于你的需求，推荐方案如下：

**技术方案：**
- 导出按钮直接放在列表顶部工具栏（"导出 Excel" + "导出 PDF" 两个按钮）
- 调用后端 `/api/export` 接口，传入当前筛选条件 + 导出格式
- 前端展示 loading，接口返回文件流后触发下载
- 文件名规则：`{页面名称}-{YYYY-MM-DD}.{ext}`
- 封装通用 `useExport` hook，各列表页面按需引入

**不做的事：**
- 不做前端本地生成文件（几万条数据性能不可接受）
- 不做异步导出 + 轮询（当前数据量级不需要）
- 不做用户自定义文件名
- 不做导出进度条

---

### 硬确认门

> 需求探索阶段完成，以下是结论摘要：
>
> - **目标**：为 ep-comp 分页列表添加数据导出能力，支持 Excel 和 PDF 两种格式
> - **范围**：做 — 后端接口对接、导出按钮、loading 状态、文件下载、通用 hook；不做 — 前端本地生成、异步轮询、自定义文件名、进度条
> - **推荐方案**：封装 `useExport` hook + 列表顶部双按钮 + 后端文件流下载
> - **核心约束**：后端已有 /api/export 接口，数据量几万条，需前端对接即可
>
> 确认后将进入 `/opsx:propose` 创建正式提案。是否继续？

### 阶段 2：结构化摘要（假设用户已确认）

**change-name：** `data-export-feature`

**proposal 内容：**
- Why：用户需要将列表数据导出为 Excel/PDF 进行离线分析或归档
- What：为 ep-comp 分页列表添加导出功能，封装通用 hook
- Scope：仅前端导出交互层，后端接口已就绪
- Impact：所有使用 ep-comp 列表的页面可按需引入导出能力

**spec 内容：**
- 列表顶部工具栏展示"导出 Excel"和"导出 PDF"两个按钮
- 点击按钮后传递当前筛选条件 + 导出格式到后端接口
- 导出期间按钮进入 loading 状态，防止重复点击
- 接口返回文件流后自动触发浏览器下载
- 文件名格式：`{页面名称}-{YYYY-MM-DD}.{ext}`
- 网络异常时提示"导出失败，请稍后重试"
- 空数据时点击导出提示"当前无数据可导出"

**design 内容：**
- 技术选型：Vue 3 Composition API + TypeScript
- 核心封装：`useExport` hook
- 参数设计：
  ```typescript
  interface ExportOptions {
    url: string              // 导出接口地址
    filename: string         // 文件名前缀
    params?: Record<string, any>  // 当前筛选条件
  }
  ```
- 接口对接：`POST /api/export`，body 包含 `format: 'excel' | 'pdf'` + 筛选参数
- 响应处理：`responseType: 'blob'`，创建 URL 触发下载

**tasks 雏形：**
1. 实现 `useExport` hook（请求封装 + blob 下载 + loading 状态）
2. 创建 `ExportButtons` 组件（Excel + PDF 双按钮）
3. 在 ep-comp 列表页面中集成导出按钮
4. 异常处理（空数据、网络错误）
5. 联调测试
